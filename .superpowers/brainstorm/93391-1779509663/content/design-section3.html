Permission System Changes

Before (current)

isSuperAdmin() = user.tenant_id === "1"
canSeePage(policy) = superadmin_only ? isSuperAdmin() : true
// Problem: every non-superadmin sees ALL non-superadmin-only pages

After (new)

// Layer determination user.role === "superadmin" → can access /superadmin/*
user.role === "senior"    → can access /senior/*
user.role === "user"      → can access /user/*

// Sidebar visibility getNavItems(role, actions) {
  if (role === "superadmin") return SUPERADMIN_NAV_ITEMS;
  if (role === "senior") return SENIOR_NAV_ITEMS;
  if (role === "user") return USER_NAV_ITEMS.filter(
    item => hasAction(item.requiredAction)
  );
}

// isSuperAdmin() replaced by: getLayer() = user.role // "superadmin" | "senior" | "user"

What changes in code

File Change
store/useAppStore.ts Add role to user type. Replace isSuperAdmin() with getLayer(). Set role cookie on login.
store/usePermissionStore.ts Replace canSeePage() with getNavItems(role, actions). Keep hasAction() for UI-level gating.
components/SidebarNav.tsx Accept navItems prop instead of hardcoding. Remove policy-based filtering (moved to nav config).
components/sidebars/*.ts NEW: 3 nav config files defining items per layer.
middleware.ts NEW: Read role cookie, enforce URL-layer match, redirect mismatches.
(superadmin)/layout.tsx NEW: SuperadminLayout with SuperadminSidebar.
(senior)/layout.tsx NEW: SeniorLayout with SeniorSidebar.
(user)/layout.tsx NEW: UserLayout with UserSidebar.
