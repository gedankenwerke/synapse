Design Section 1: Auth & Middleware

Cookie-based role enforcement with server-side guards

Auth Data Flow
Login Response
POST /login →
{
  user: { id, username,
    tenant_id, role,
    created_at, updated_at },
  access_token: "eyJ...",
  refresh_token: "eyJ...",
  token_type: "Bearer",
  expires_in: 900
}
Cookies Set (client-side)
auth_token=eyJ...
  httpOnly=true
  path=/

refresh_token=eyJ...
  httpOnly=true
  path=/

role=superadmin|senior|user
  httpOnly=true
  path=/
Zustand Store (client-side)
useAppStore:
  user: { id, username,
    tenant_id, role,
    created_at, updated_at }
  token: "eyJ..."
  refreshToken: "eyJ..."
  isAuthenticated: true
Middleware Logic (middleware.ts)
const role = request.cookies.get('role')?.value; const token = request.cookies.get('auth_token')?.value; const locale = extractLocale(pathname); // No token → login if (!token) redirect(`/${locale}/`); // No role → try /me to get role, set cookie, retry if (!role) { /* call /me, set role cookie, continue */ } // Role mismatch → redirect to correct layer const layerPath = `/${locale}/${role}`; if (!pathname.startsWith(layerPath)) { redirect(`${layerPath}/dashboard`); } // Role matches → pass through next();
